GitHub Setup Guide (~10 min)

  1. Create a public GitHub repo — github.com/new → name: yt-shorts-bot → set Public (= unlimited free minutes) → Create repository.
  2. Push the kit files — Extract the downloaded ZIP. Push all files to your repo. The .github/workflows/shorts.yml file is what triggers the schedule.
  3. YouTube OAuth (once, on your computer) — Go to Google Cloud Console → create project → enable YouTube Data API v3 → Credentials → Create OAuth 2.0 Client ID (type: Desktop app) → Download JSON → rename to client_secrets.json. Then run: pip install -r requirements.txt then python setup_auth.py. A browser opens, sign in with your YouTube channel → copy the printed JSON.
  4. Add 4 GitHub Secrets — Your repo → Settings → Secrets and variables → Actions → New secret:
    • GEMINI_API_KEY — from Google AI Studio
    • PEXELS_API_KEY — from Pexels
    • YOUTUBE_TOKEN_JSON — full JSON from step above
    • DEFAULT_NICHE — e.g. motivation
  5. Test it now — Go to your repo's Actions tab → "🎬 YouTube Shorts Automator" → "Run workflow" → Run. Wait ~5 minutes → check your YouTube channel.
  6. Done — it runs forever. GitHub's servers create and upload 2–3 Shorts per day at your scheduled times. Phone off, laptop off — doesn't matter.
